Constants Reference

$TFW_COLOR_*

These constants are single-character prefix strings recognized by SciTE's output pane to color console lines. They are used as the optional $sColor parameter of _TestFmkHeader(), and internally by _TestFmkAssert() and _TestFmkSummary() to color output lines automatically.

$TFW_COLOR_WHITE ":" White (no color highlight). Used for the Failed line when count is zero.
$TFW_COLOR_RED "!" Bold red. Used for [FAIL] lines and the Failed count when non-zero.
$TFW_COLOR_BLUE ">" Blue. Used for [PASS] lines and the Passed count.
$TFW_COLOR_YELLOW "-" Bold yellow. Available for use with _TestFmkHeader() to distinguish test sections visually.
$TFW_COLOR_ORANGE "+" Bold orange. The default color for _TestFmkHeader() and the Total count in the summary.

Remarks

These prefix characters are prepended to the text passed to ConsoleWrite(). SciTE maps each character to a color based on its style.errorlist.* properties. The exact colors rendered depend on the user's SciTE theme/configuration, but the defaults described above match the standard SciTE4AutoIt3 install.

These constants are only meaningful when the script is run from SciTE (via Go/F5 or a Tools menu command). When run from a plain command prompt, the prefix character is printed as a literal character before the text and has no visual significance.

Example

#include <TestFramework.au3>

; Use $TFW_COLOR_YELLOW to distinguish a sub-group of tests visually
_TestFmkHeader("Test: Setup", $TFW_COLOR_YELLOW)

; Use the default orange for the main test group
_TestFmkHeader("Test: Main assertions")
