Getting Started

Installation

If you installed AutoIt Test Framework via the provided installer, it is placed in AutoIt's standard include directory and can be referenced from any project using the angle-bracket form:

#include <TestFramework.au3>

The installer also places Testable.au3 and Stubs.au3 in the same directory, so they are available the same way:

#include <Testable.au3>
#include <Stubs.au3>

If you are using the files without installing them (for example, by placing them alongside your own scripts), use the quoted form with a relative or full path:

#include "TestFramework.au3"

Testable.au3 and Stubs.au3 are opt-in libraries for testing code that uses GUI controls, dialogs, file system, registry, and other side-effecting AutoIt built-ins. See the Testable Reference and Stubs Reference for details.

Anatomy of a Test File

A typical test file using AutoIt Test Framework follows this structure:

1. Include the framework and the code under test

#include <TestFramework.au3>
#include "MyLibrary.au3"

2. Write one function per test group

Each test function calls _TestFmkHeader() once at the top to label the group, then one or more _TestFmkAssert() calls to verify behavior. Always pass the actual value and the expected value as the 3rd and 4th arguments so that failures show exactly what went wrong. 

Func _TestAdd()
    _TestFmkHeader("Test: Add()")
    _TestFmkAssert(Add(1, 2) = 3,   "Add(1, 2) returns 3",   Add(1, 2),   "3")
    _TestFmkAssert(Add(-1, 1) = 0,  "Add(-1, 1) returns 0",  Add(-1, 1),  "0")
    _TestFmkAssert(Add(0, 0) = 0,   "Add(0, 0) returns 0",   Add(0, 0),   "0")
EndFunc

3. Wire tests into _RunAllTests() using _TestFmkRun()

_TestFmkRun() runs a test function and feeds its pass/fail result into a cumulative $bAllPassed variable. This means even if an early test fails, all remaining tests still run and are reported. 

Func _RunAllTests()
    Local $bAllPassed = True
    $bAllPassed = _TestFmkRun(_TestAdd,      $bAllPassed)
    $bAllPassed = _TestFmkRun(_TestSubtract, $bAllPassed)
    $bAllPassed = _TestFmkRun(_TestMultiply, $bAllPassed)
    _TestFmkSummary()
    Return $bAllPassed
EndFunc

_RunAllTests()

4. Call _TestFmkSummary() at the end

_TestFmkSummary() prints the final Total/Passed/Failed block to the console. Call it once at the end of _RunAllTests(), not inside individual test functions.

Complete Worked Example

This example tests a small custom math library, showing a realistic test file from start to finish.

MathLib.au3 (the code under test)

; A simple math library
Func Add($a, $b)
    Return $a + $b
EndFunc

Func Divide($a, $b)
    If $b = 0 Then Return SetError(1, 0, 0)
    Return $a / $b
EndFunc

MathLibTests.au3 (the test file)

#include <TestFramework.au3>
#include "MathLib.au3"

Func _TestAdd()
    _TestFmkHeader("Test: Add()")
    _TestFmkAssert(Add(2, 3) = 5,   "Add(2, 3) returns 5",   Add(2, 3),   "5")
    _TestFmkAssert(Add(-1, 1) = 0,  "Add(-1, 1) returns 0",  Add(-1, 1),  "0")
    _TestFmkAssert(Add(0, 0) = 0,   "Add(0, 0) returns 0",   Add(0, 0),   "0")
EndFunc

Func _TestDivide()
    _TestFmkHeader("Test: Divide()")
    _TestFmkAssert(Divide(10, 2) = 5,  "Divide(10, 2) returns 5",  Divide(10, 2),  "5")
    _TestFmkAssert(Divide(7, 2) = 3.5, "Divide(7, 2) returns 3.5", Divide(7, 2),   "3.5")

    ; Test divide by zero - expect @error to be set
    Divide(1, 0)
    _TestFmkAssert(@error = 1, "Divide(1, 0) sets @error = 1", @error, "1")
EndFunc

Func _RunAllTests()
    Local $bAllPassed = True
    $bAllPassed = _TestFmkRun(_TestAdd,    $bAllPassed)
    $bAllPassed = _TestFmkRun(_TestDivide, $bAllPassed)
    _TestFmkSummary()
    Return $bAllPassed
EndFunc

_RunAllTests()

Console output (all passing)

> [PASS] Add(2, 3) returns 5
> [PASS] Add(-1, 1) returns 0
> [PASS] Add(0, 0) returns 0

+ --- Test: Divide() ---
> [PASS] Divide(10, 2) returns 5
> [PASS] Divide(7, 2) returns 3.5
> [PASS] Divide(1, 0) sets @error = 1

+ Total: 6
> Passed: 6

Console output (one failure)

Testing Scripts with GUI and User Interaction

AutoIt Test Framework also supports testing scripts that use GUI controls, dialogs, file system, registry, and other side-effecting built-ins through the Testable.au3 and Stubs.au3 libraries.

The pattern has two sides. In script code, use _Tstbl_* wrappers instead of AutoIt built-ins directly. In test files, include Stubs.au3 which automatically rewires all wrappers to stubs that record calls and return controlled values.

Script code (FileCleanup.au3)

#include <Testable.au3>
#include <MsgBoxConstants.au3>

If Not IsDeclared("__FILECLEANUP_TEST_MODE") Then
    _Main()
EndIf

Func _Main()
    Local $sPath = "C:\temp\myfile.txt"
    If DeleteFileWithConfirmation($sPath) Then
        ConsoleWrite("File deleted." & @CRLF)
    EndIf
EndFunc

Func DeleteFileWithConfirmation($sPath)
    If Not _Tstbl_FileExists($sPath) Then
        _Tstbl_MsgBox($MB_OK + $MB_ICONERROR, "File Cleanup", "File not found:" & @CRLF & $sPath)
        Return False
    EndIf

    Local $iResult = _Tstbl_MsgBox($MB_YESNO + $MB_ICONQUESTION, "File Cleanup", _
        "Are you sure you want to delete:" & @CRLF & $sPath & "?")

    If $iResult = $IDYES Then
        _Tstbl_FileDelete($sPath)
        _Tstbl_MsgBox($MB_OK + $MB_ICONINFORMATION, "File Cleanup", "File deleted successfully.")
        Return True
    EndIf

    _Tstbl_MsgBox($MB_OK + $MB_ICONINFORMATION, "File Cleanup", "Deletion cancelled.")
    Return False
EndFunc

Test file (FileCleanupTests.au3)

Global $__FILECLEANUP_TEST_MODE = True
#include <TestFramework.au3>
#include <Stubs.au3>
#include "FileCleanup.au3"

Func _TestDeleteFileWithConfirmation_FileNotFound()
    _TestFmkHeader("Test: DeleteFileWithConfirmation() - file not found")

    _ResetStubs()
    _SetStubReturn("FileExists", 1, False)

    Local $bResult = DeleteFileWithConfirmation("C:\temp\myfile.txt")

    _TestFmkAssert($bResult = False, "Returns False when file not found", $bResult, False)
    _TestFmkAssert($g_StubCalls["MsgBox"].count = 1, "Shows one MsgBox", $g_StubCalls["MsgBox"].count, 1)
    _TestFmkAssert(Not MapExists($g_StubCalls, "FileDelete"), "FileDelete not called", MapExists($g_StubCalls, "FileDelete"), False)
EndFunc

Func _TestDeleteFileWithConfirmation_UserConfirms()
    _TestFmkHeader("Test: DeleteFileWithConfirmation() - user confirms")

    _ResetStubs()
    _SetStubReturn("FileExists", 1, True)
    _SetStubReturn("MsgBox", 1, $IDYES)

    Local $bResult = DeleteFileWithConfirmation("C:\temp\myfile.txt")

    _TestFmkAssert($bResult = True, "Returns True when confirmed", $bResult, True)
    _TestFmkAssert($g_StubCalls["FileDelete"].count = 1, "FileDelete called once", $g_StubCalls["FileDelete"].count, 1)
    _TestFmkAssert($g_StubCalls["FileDelete"][1]["sPath"] = "C:\temp\myfile.txt", _
        "Correct file deleted", $g_StubCalls["FileDelete"][1]["sPath"], "C:\temp\myfile.txt")
EndFunc

Func _TestDeleteFileWithConfirmation_UserCancels()
    _TestFmkHeader("Test: DeleteFileWithConfirmation() - user cancels")

    _ResetStubs()
    _SetStubReturn("FileExists", 1, True)
    _SetStubReturn("MsgBox", 1, $IDNO)

    Local $bResult = DeleteFileWithConfirmation("C:\temp\myfile.txt")

    _TestFmkAssert($bResult = False, "Returns False when cancelled", $bResult, False)
    _TestFmkAssert(Not MapExists($g_StubCalls, "FileDelete"), "FileDelete not called", MapExists($g_StubCalls, "FileDelete"), False)
EndFunc

Func _RunAllTests()
    Local $bAllPassed = True
    $bAllPassed = _TestFmkRun(_TestDeleteFileWithConfirmation_FileNotFound, $bAllPassed)
    $bAllPassed = _TestFmkRun(_TestDeleteFileWithConfirmation_UserConfirms, $bAllPassed)
    $bAllPassed = _TestFmkRun(_TestDeleteFileWithConfirmation_UserCancels,  $bAllPassed)
    _TestFmkSummary()
    Return $bAllPassed
EndFunc

_RunAllTests()
Note: _SetStubReturn("MsgBox", 1, $IDYES) pre-configures what the first MsgBox call returns. $g_StubCalls["MsgBox"].count returns how many times it was called. $g_StubCalls["FileDelete"][1]["sPath"] reads the sPath argument of the first FileDelete call. Stub indexes are 1-based. Call _ResetStubs() between tests to clear all recorded calls and returns. To assert a stub was never called use Not MapExists($g_StubCalls, "TypeName") rather than checking .count = 0, since if a stub was never called its map entry does not exist yet.

Compiled scripts and #pragma compile

Scripts that use #RequireAdmin trigger a UAC prompt at interpreted runtime, which breaks testing. Replace it with the compile-time directive instead: 

#pragma compile(ExecLevel, requireAdministrator)

This applies the elevation requirement only to the compiled .exe, not to the interpreted script, so tests run without a UAC prompt.

FileInstall in compiled scripts

FileInstall cannot be called through a function pointer because Aut2Exe requires a string literal as the source path at compile time to embed the file into the exe. The workaround is a wrapper function with a Select block containing all literal FileInstall calls. Aut2Exe finds the literals at compile time, and the wrapper can be stubbed in tests via a function pointer: 

Func __FileInstall($sSrc, $sDest, $iFlag)
    Select
        Case $sSrc = "path\to\file1.au3"
            FileInstall("path\to\file1.au3", $sDest, $iFlag)
        Case $sSrc = "path\to\file2.au3"
            FileInstall("path\to\file2.au3", $sDest, $iFlag)
    EndSelect
EndFunc

Global $g_hFn_FileInstall = __FileInstall

See the Testable Reference and Stubs Reference for the full list of supported wrappers and stubs.

Tips

Always pass actual and expected values to _TestFmkAssert(). The 3rd and 4th arguments are optional, but without them a failure only tells you the description failed, not what the actual result was. Always include them.

Keep test functions focused. One test function per unit of behavior. If a function has 20 assertions, consider splitting it into smaller, more specific test functions with clearer headers.

Name test functions descriptively. _TestAdd_NegativeNumbers() is more useful than _TestAdd2() when reading the console output after a failure.

Test both the happy path and the error path. As shown in the _TestDivide() example above, testing that @error is correctly set on failure is just as important as testing correct return values on success. Remember: capture @error into a local variable immediately after the call, since any subsequent function call resets it.

Write tests before fixing bugs (TDD). When a test fails because a stub was called fewer times than expected, accessing $g_StubCalls["TypeName"][index]["property"] on a non-existent index returns an empty string gracefully rather than crashing the script. This means you can write tests against code that does not yet do what it should, and the test suite will report failures cleanly without breaking the test run.
