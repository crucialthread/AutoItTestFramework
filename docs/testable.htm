Testable Reference

Overview

Testable.au3 provides wrapper functions for AutoIt built-ins that have side effects or return values that affect program flow. Each wrapper routes the call through a function pointer global ($g_hFn_*), which in script code points to the real AutoIt built-in, and in test code is rewired by Stubs.au3 to a stub that records calls and returns controlled values.

This allows tests to verify that the correct functions were called with the correct parameters, and to control what those functions return, without any real GUI being created, no files being written or deleted, and no registry entries being touched.

Usage

Include Testable.au3 in your script code and use _Tstbl_* wrappers instead of AutoIt built-ins directly:

#include <Testable.au3>

Func DeleteFileWithConfirmation($sPath)
    If Not _Tstbl_FileExists($sPath) Then
        _Tstbl_MsgBox($MB_OK + $MB_ICONERROR, "Cleanup", "File not found.")
        Return False
    EndIf
    Local $iResult = _Tstbl_MsgBox($MB_YESNO + $MB_ICONQUESTION, "Cleanup", "Delete " & $sPath & "?")
    If $iResult = $IDYES Then
        _Tstbl_FileDelete($sPath)
        Return True
    EndIf
    Return False
EndFunc

In test files, include Stubs.au3 instead of Testable.au3. Stubs.au3 includes Testable.au3 automatically and rewires all function pointers to stubs. See the Stubs Reference for details.

You can also include individual category files directly instead of the full Testable.au3, for example #include <Testable_Dialogs.au3>, if you only need wrappers for a specific category.

Remarks

All _Tstbl_* wrappers propagate @error and @extended from the underlying call back to the caller using SetError(@error, @extended, $vResult). This means code that checks @error after a built-in call will work correctly when using the testable wrapper.

Note: Testable.au3 is an umbrella file that includes all Testable_*.au3 category files. The category files can also be included individually if only a subset of wrappers is needed.

Wrapper Functions

All _Tstbl_* wrappers accept exactly the same parameters as their AutoIt built-in equivalents and behave identically in script code. For parameter details and usage refer to the AutoIt documentation.

Special case: FileInstall cannot be routed through a function pointer and requires a different approach to be testable. See FileInstall Special Case for details.
