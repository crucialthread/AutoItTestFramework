Stubs Reference

Overview

Stubs.au3 provides stub implementations for all _Tstbl_* wrappers defined in Testable.au3. Including Stubs.au3 in a test file automatically rewires all function pointer globals to stubs - no manual setup needed.

Each stub serves two purposes:

  • Call recording - every call is recorded with its parameters into $g_StubCalls so tests can verify that the correct functions were called with the correct arguments.
  • Return control - tests can pre-configure what a stub returns for each call via _SetStubReturn(), allowing tests to drive the code under test down specific branches.

Usage

Include Stubs.au3 in test files. It automatically includes Testable.au3 and rewires all function pointers:

Global $__MY_SCRIPT_TEST_MODE = True
#include <TestFramework.au3>
#include <Stubs.au3>
#include "MyScript.au3"

Func _TestSomething()
    _TestFmkHeader("Test: something")

    _ResetStubs()
    _SetStubReturn("MsgBox", 1, $IDYES)

    MyFunction()

    _TestFmkAssert($g_StubCalls["MsgBox"].count = 1, "MsgBox called once", $g_StubCalls["MsgBox"].count, 1)
    _TestFmkAssert($g_StubCalls["MsgBox"][1]["sTitle"] = "Confirm", "MsgBox title correct", $g_StubCalls["MsgBox"][1]["sTitle"], "Confirm")
EndFunc

You can also include individual category stub files directly, for example #include <Stubs_Dialogs.au3>. Each category file includes Stubs_Core.au3 automatically.

Functions

_ResetStubs()

_ResetStubs()

Clears all recorded calls in $g_StubCalls and all configured return values in $g_StubReturns. Call this at the start of each test to ensure a clean state. 

_ResetStubs()
_SetStubReturn("FileExists", 1, True)

MyFunction()

_TestFmkAssert($g_StubCalls["FileExists"].count = 1, ...)

_SetStubReturn($sType, $iIdx, $vValue)

_SetStubReturn($sType, $iIdx, $vValue)

Pre-configures the return value for the Nth call of a given stub type. Call this before running the code under test to control which branches execute.

$sTypeThe stub type name as a string, e.g. "MsgBox", "FileExists", "RegRead".
$iIdxThe 1-based call index. 1 configures the return for the first call, 2 for the second, and so on.
$vValueThe value to return for that call. Use "__ERROR__" for stubs that support it (e.g. RegRead) to simulate a function setting @error.
; First MsgBox returns YES, second returns NO
_SetStubReturn("MsgBox", 1, $IDYES)
_SetStubReturn("MsgBox", 2, $IDNO)

; Simulate RegRead failing with @error
_SetStubReturn("RegRead", 1, "__ERROR__")

If no return is configured for a given call index the stub falls back to a sensible default (e.g. $IDOK for MsgBox, True for FileExists, empty string for RegRead).

Stub Stores

$g_StubCalls

A map keyed by stub type name. Each entry is itself a map of recorded calls keyed by 1-based call index, plus a .count property holding the total number of calls. 

; Check how many times FileDelete was called
$g_StubCalls["FileDelete"].count

; Read the sPath argument of the first FileDelete call
$g_StubCalls["FileDelete"][1]["sPath"]

; Read the sTitle argument of the second MsgBox call
$g_StubCalls["MsgBox"][2]["sTitle"]
Note: If a stub was never called its entry does not exist in $g_StubCalls. Accessing .count or a call index on a non-existent entry will cause a runtime error. Use MapExists($g_StubCalls, "TypeName") to check whether a stub was ever called before accessing its data. 
; Correct way to assert a stub was never called
_TestFmkAssert(Not MapExists($g_StubCalls, "FileDelete"), "FileDelete not called", MapExists($g_StubCalls, "FileDelete"), False)

$g_StubReturns

A map keyed by stub type name. Each entry is a map of return values keyed by 1-based call index. Populated by _SetStubReturn() before running the code under test.

Recorded Call Properties

Each stub records the parameters passed to it as named properties on the call object.

Special case: FileInstall requires a different approach to be testable, as such, there is no testable either stub for this function. See FileInstall Special Case for details.

Remarks

1-based indexing. Call indexes start at 1. The first call to a stub is index 1, the second is index 2, and so on. $g_StubCalls["MsgBox"].count reflects the total number of calls and also equals the index of the last call.

Checking for stubs never called. If a stub was never called its entry does not exist in $g_StubCalls. Use MapExists($g_StubCalls, "TypeName") to check safely rather than accessing .count directly, which would cause a runtime error.

Simulating @error. For stubs that support it (currently RegRead), pass "__ERROR__" as the return value to _SetStubReturn(). The stub will call SetError(1) before returning, simulating a failed registry read.

FileInstall. FileInstall requires a different approach to be testable, as such, there is no testable either stub for this function. See FileInstall Special Case for details.
