FileInstall Special Case

The Problem

FileInstall cannot be routed through a function pointer like other testable wrappers. Aut2Exe performs a static analysis pass over the source code at compile time, looking for FileInstall("literal string", ...) patterns to know which files to embed into the compiled exe. When FileInstall is called through a variable, Aut2Exe does not find it and the file is not embedded - causing the compiled exe to fail at runtime.

The Solution

Create a __FileInstall wrapper function with a Select block containing all literal FileInstall calls. Aut2Exe finds the literal calls inside the Select block at compile time and embeds all files correctly. A function pointer global is then assigned to this wrapper rather than to FileInstall directly, which allows the pointer to be replaced with a stub in tests. 

; $g_hFn_FileInstall points to __FileInstall (a wrapper function) rather than FileInstall directly.
; This allows stubbing in tests while keeping literal FileInstall calls visible to Aut2Exe for embedding.
Global $g_hFn_FileInstall = __FileInstall

Func __FileInstall($sSrc, $sDest, $iFlag)
    Select
        Case $sSrc = "path\to\file1.au3"
            FileInstall("path\to\file1.au3", $sDest, $iFlag)
        Case $sSrc = "path\to\file2.au3"
            FileInstall("path\to\file2.au3", $sDest, $iFlag)
        Case $sSrc = "path\to\file3.exe"
            FileInstall("path\to\file3.exe", $sDest, $iFlag)
    EndSelect
EndFunc

In script code, call through the function pointer instead of calling FileInstall directly:

$g_hFn_FileInstall("path\to\file1.au3", $sDestFolder & "\file1.au3", $FC_OVERWRITE)
$g_hFn_FileInstall("path\to\file2.au3", $sDestFolder & "\file2.au3", $FC_OVERWRITE)

Stubbing in Tests

In test files, declare a stub function that records calls and assign it to $g_hFn_FileInstall after including the script under test: 

Global $__MY_SCRIPT_TEST_MODE = True
#include <TestFramework.au3>
#include <Stubs.au3>
#include "MyScript.au3"

; Stub wired after the installer include since $g_hFn_FileInstall is declared inside it
Func _Stub_FileInstall($sSrc, $sDest, $iFlag)
    __StubInitType("FileInstall")
    Local $iIdx = $g_StubCalls["FileInstall"].count + 1
    Local $oCall[]
    $oCall["sSrc"]  = $sSrc
    $oCall["sDest"] = $sDest
    $g_StubCalls["FileInstall"][$iIdx] = $oCall
    $g_StubCalls["FileInstall"].count  = $iIdx
EndFunc

$g_hFn_FileInstall = _Stub_FileInstall

; Now tests can verify FileInstall calls
Func _TestMyInstall()
    _TestFmkHeader("Test: install copies files")

    _ResetStubs()
    $g_hFn_FileInstall = _Stub_FileInstall

    __RunInstall(1, 2)

    _TestFmkAssert($g_StubCalls["FileInstall"].count = 3, "Installs 3 files", $g_StubCalls["FileInstall"].count, 3)
    _TestFmkAssert($g_StubCalls["FileInstall"][1]["sSrc"] = "path\to\file1.au3", _
        "First file correct", $g_StubCalls["FileInstall"][1]["sSrc"], "path\to\file1.au3")
EndFunc
Note: _ResetStubs() only clears $g_StubCalls and $g_StubReturns - it does not reset function pointer globals. This means $g_hFn_FileInstall is not handled by _ResetStubs() and must be managed individually in each test that needs it.
